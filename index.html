Git Guide: Commands, Concepts, and Scenarios

1. Initializing a Repository

To start a new repository, use:

git init

This initializes a new Git repository in the current directory.

2. Checking Repository Status

View the current status of the repository with:

git status

This shows modified files, untracked files, and branch information.

3. Committing Changes

To stage and commit changes:

git add .
git commit -m "Commit message"

The first command stages all changes, and the second commits them.

4. Connecting to a Remote Repository

To add a remote repository:

git remote add origin <repository-url>

Verify the remote URL with:

git remote -v

5. Pushing Changes

Push changes to the remote repository:

git push -u origin main

The -u flag sets the upstream branch.

6. Viewing Commit History

See a log of commits with:

git log --oneline

This shows a condensed list of commit hashes and messages.

7. Reverting a Commit

To undo a commit:

git revert <commit-hash>

This creates a new commit that undoes the specified commit.

8. Setting Up Multiple GitHub Accounts

To manage multiple GitHub accounts, configure the SSH settings in ~/.ssh/config:

Host github-personal
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa_personal

Host github-work
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa_work

Clone repositories using:

git clone git@github-personal:username/repo.git

9. Resolving Authentication Issues

If authentication fails, check SSH keys with:

ssh -T git@github.com

Ensure the correct SSH key is being used.

Conclusion

This guide covers essential Git commands and scenarios. With these, you can confidently manage repositories, resolve issues, and work efficiently with Git.
