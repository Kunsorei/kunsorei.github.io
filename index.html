Git & SSH: Complete Guide

A detailed guide on Git commands, SSH configuration, managing multiple GitHub accounts, and troubleshooting common issues.

1. Initializing a Git Repository

To start tracking files in a project, initialize a Git repository:

git init

This creates a .git directory in the project folder.

2. Viewing Git Commit History

Check commit history to find commit hashes:

git log --oneline

3. Reverting Changes

To undo a commit, find the commit hash from git log and run:

git revert <commit-hash>

4. Understanding HEAD and Reflog

HEAD represents the latest commit. To see previous states:

git reflog

Example output:

58bb193 (HEAD -> main) HEAD@{0}: revert: eevert "add style"
b6becb3 (origin/main) HEAD@{1}: commit: add style

5. Handling Upstream Branch Issues

If you see an upstream branch error:

git push --set-upstream origin main

This links your local branch to the remote branch.

6. Managing Multiple GitHub Accounts

If you work with multiple GitHub accounts, configure SSH keys:

# Personal GitHub Account
Host github.com-alikojiro
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_rsa
  IdentitiesOnly yes

# Work GitHub Account
Host github.com-kunsorei
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_git
  IdentitiesOnly yes

7. Switching Between Accounts

Update your remote URL to match the SSH config:

git remote set-url origin git@github.com-kunsorei:Kunsorei/repo.git

Verify authentication:

ssh -T git@github.com-kunsorei

8. Pushing Changes After Initializing Git

After running git init, add a remote repo and push changes:

git remote add origin git@github.com-alikojiro:username/repo.git
git push -u origin main

9. Troubleshooting Permission Errors

If you see an error like:

ERROR: Permission to Kunsorei/repo.git denied

Ensure you are using the correct SSH key:

ssh -T git@github.com-kunsorei

For one-time use of a specific SSH key:

GIT_SSH_COMMAND="ssh -i ~/.ssh/id_ed25519_git" git push origin main

To set it permanently for this repository:

git config core.sshCommand "ssh -i ~/.ssh/id_ed25519_git"

With this guide, you can confidently manage Git repositories and multiple GitHub accounts using SSH.
